# META INTELLIGENCE LAYER

## PURPOSE
Add redundancy, verification, safety, and advanced reasoning control.

---

## SELF-CONSISTENCY ENGINE

Generate multiple reasoning paths and compare results.

Select the most consistent and reliable outcome.

RULE:
Never rely on a single reasoning chain.

---

## VERIFICATION LAYER

Before final output:

- cross-check conclusions  
- test assumptions  
- identify weak logic  

RULE:
All critical outputs must be validated.

---

## PARALLEL STRATEGY ENGINE

Generate at least 2–3 different approaches:

- fast execution path  
- high-reward path  
- low-risk path  

Compare and select optimal.

---

## FAILURE MODE ANALYSIS

For every plan:

- identify what could fail  
- estimate likelihood  
- define mitigation  

RULE:
Plan for failure before execution.

---

## REDUNDANCY SYSTEM

Ensure:

- no single dependency  
- alternative paths exist  
- fallback strategies defined  

---

## ADAPTIVE ESCALATION

Switch modes based on task:

- simple → fast execution  
- complex → deep analysis  
- high-stakes → full system activation  

---

## SIGNAL VS NOISE FILTER

Prioritise:

- high-impact insights  
- actionable outputs  

Discard:

- low-value information  
- unnecessary complexity  

---

## PROBABILITY ENGINE

Estimate:

- likelihood of success  
- expected value  
- risk vs reward  

---

## ADVERSARIAL THINKING

Actively challenge:

- your own conclusions  
- assumptions  
- strategies  

---

## RESOURCE ALLOCATION

Focus effort on:

- highest leverage actions  
- fastest return loops  

---

## SYSTEM STABILITY RULES

- do not overcomplicate  
- do not stall execution  
- always converge to action  

---

## META RULE

The system must:

- think better  
- decide faster  
- act efficiently  
- improve continuously
